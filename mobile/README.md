# 뛰지마 실내 · Capacitor (Android only)

Android shell that wraps `pipeline/review_ui/user.html` + `src/location/*.js`
with a real Wi-Fi scanner. iOS is out of scope for now — Apple restricts the
public Wi-Fi scan API to carriers, so fingerprint matching can't run on iPhone
without a special entitlement.

## Layout

```
mobile/
├── package.json
├── capacitor.config.json
├── scripts/
│   └── bundle-data.mjs    # packs data/output + UI into www/
├── www/
│   ├── index.html         # boot screen, sets up native bridge, opens user.html
│   ├── user.html          # copied from pipeline/review_ui/ by bundle-data
│   ├── src/
│   │   ├── native-bridge.js   # rewrites /api/* → ./data/*, exposes nativeScan
│   │   └── location/*.js      # copied from src/location
│   └── data/              # generated offline bundle (graphs, fingerprints, images)
└── android/               # generated by `npx cap add android`
```

## Setup (first time)

Using **Capacitor 8** + **@capgo/capacitor-wifi v8** (the `@capawesome-team`
package the earlier draft named does not exist — that was my mistake).

```bash
cd mobile
npm install
npm run prepare:data            # build www/data/ from ../data/output
npx cap init "뛰지마 실내" com.dontrun.indoor --web-dir www
npx cap add android
npm run sync
```

## Dev loop

```bash
# After editing pipeline/review_ui/user.html or src/location/*.js:
npm run sync                    # rebundles data + copies UI + cap sync

# Open in Android Studio:
npm run open:android

# Or run straight on a connected device:
npm run run:android
```

## Permissions

`android/app/src/main/AndroidManifest.xml` needs:

```xml
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.ACCESS_WIFI_STATE" />
<uses-permission android:name="android.permission.CHANGE_WIFI_STATE" />
<uses-permission android:name="android.permission.NEARBY_WIFI_DEVICES"
  android:usesPermissionFlags="neverForLocation" />
```

Android 10+ throttles Wi-Fi scans to 4 per 2 minutes for foreground apps and
effectively 0 for background. That's fine for tap-to-scan UX.

## How detection works on device

1. User taps 📍 in `user.html`.
2. `runDetection()` sees `window.hasNativeScan === true` and calls
   `window.nativeScan()` (injected by `index.html` into the iframe).
3. `native-bridge.js` calls `Wifi.scanWifi()` via
   `@capawesome-team/capacitor-wifi`, normalises to `{bssid, rssi}[]`.
4. `WifiPositioning.match(scans)` runs the same KNN as the desktop
   simulation, returns `{lat, lng, floor, confidence, ...}`.
5. Georef transform maps lat/lng → canvas coords, snaps to nearest graph
   node, and the routing UI takes over.

## Known limitations

- iOS not supported (Apple API lockout).
- BSSID randomisation on some routers can degrade matching; the fingerprint
  corpus was collected before MAC randomisation became common, so some
  Android devices may need `PRIVACY_RANDOMIZED_MAC` disabled per-network to
  match reliably. Not addressed here.
- Data bundle is baked at build time — changing a station requires
  `npm run sync`.
